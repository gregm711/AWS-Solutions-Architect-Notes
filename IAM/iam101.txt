What does IAM give you:
	Centralized control of AWS Account
	Shared access to AWS account
	Granular Permissions
	Identity federation (facebook, linkedin)
	Multifactor Authentication
	Temprary access for users when neccessary
	password rotation policy
	Integrates with other AWS services
	supports PCI DSS compliance

Critical Terms:
	Users
	Groups - collection of users under one set of permissions
	Roles - determine what users can and cannot access
	Policies - A document that defines one or more permissions

Summary:
	IAM is global,
	new users have no permissions when created
	new users are assigned Acces key ID & Secret Access Key on creation
	setup multifactor Authentication on root
	custom password rotation policies
